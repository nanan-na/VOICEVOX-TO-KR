# 한국어 → VOICEVOX 합성 도구

한국어 텍스트를 자체 규칙엔진으로 가타카나+악센트 마크업으로 변환해, 로컬 VOICEVOX ENGINE(`localhost:50021`)으로 발음시키는 개인용 도구 (CLI + 웹 UI). 코드 주석의 "기획서 x.x"는 별도로 관리하는 설계 문서의 장 번호입니다.

## 설치

1. [Node.js](https://nodejs.org/) LTS 설치
2. [VOICEVOX](https://voicevox.hiroshiba.jp/) 설치 (무료)
3. 이 저장소를 `Code → Download ZIP`으로 받아 압축 해제 (또는 `git clone`)
4. `start.bat` 더블클릭 — 엔진 확인 → 서버 → 브라우저가 자동으로 열린다

외부 npm 패키지가 없어서 `npm install`은 필요 없습니다. 막약 VOICEVOX를 기본 경로가 아닌 곳에
설치했다면 VOICEVOX를 먼저 실행해 두면, `start.bat`이 그대로 이어받습니다.

## 웹 UI (Phase 3)

`start.bat` 더블클릭 한 번으로 엔진 확인 → 서버 → 브라우저까지 열린다.
(수동 실행: `node server.js` 후 `http://127.0.0.1:8300`)

- **버튼식 변환**: [변환] 또는 Ctrl+Enter로만 변환 (자동 변환 없음). `☑ 자동 재생`이면 변환 직후 재생.
  입력을 고치면 `⚠ 입력 변경됨` 배너가 뜨고 결과가 흐려진다 — 결과 상단에 `변환 완료 ✔ 시각` 표시.
- **어절별 병기 + 인라인 교정**: 결과의 어절을 클릭하면 팝오버에서
  `원본 변환(규칙) / 현재 사전값 / 새 값`을 비교하며 교정 → [저장 및 재생].
  저장처는 audition과 같은 마커 맥락 라우팅(`*`→dict_jp, 한글→dict, 영어→dict_en).
- **재생 중 어절 하이라이트**: 지금 발음되는 어절이 병기 테이블에서 순서대로 강조된다.
- **가나 마크업 직접 편집**: 자동 변환된 마크업을 고쳐 [편집한 가나로 재생]
  (수정 후 재생 전엔 `● 미적용` 배지). 가상 가나 키보드(한글 발음 라벨) 지원.
- **사전 브라우저**: dict/dict_en/dict_jp 3권 탭 — 검색·수정·삭제·직접 추가.
- **화자 선택**: 같은 화자의 스타일은 서브메뉴로 묶이고, 화자·스타일마다 프로필 아이콘이 표시된다
  (엔진 `speaker_info`에서 자동 수집). 선택·JP 토글·자동 재생 설정은 `config.json`에 유지된다.

## CLI 사용법

VOICEVOX(ENGINE 포함)를 먼저 실행한 뒤:

```
node k2v.js "안녕하세요"                        # → out.wav
node k2v.js "뭐 해?" --out hi.wav --speaker 3   # 화자·출력 지정
node k2v.js "까치" --dry                        # 변환 결과만 (엔진 불필요)
node k2v.js "meeting 하자" --dry                # 영어 → 콩글리시 발음 (3.7)
node k2v.js "커피* 한잔" --dry                  # * 마커 → 그 단어만 일본어식 (コオヒイ)
node k2v.js "커피 마시자" --jp --dry            # 전역 일본어식 토글
node k2v.js --speakers                          # 화자 목록
```

영어 단어는 `dict_en.json`(관용 표기) → 약어 letter-name → 번들 발음사전(`data/cmudict.json`,
CMUdict) + 외래어 표기법 순으로 한글로 바꿔 기존 규칙 파이프라인에 태운다. 일본어식(`*`/`--jp`)은
한글 단어는 `dict_jp.json` 조회, 영어 단어는 CMUdict→가타카나 자동 변환.

## 청취·사전 축적 도구

발음을 귀로 확인하며 예외사전(dict.json)을 채우는 대화형 CLI (엔진 필요):

```
node tools/audition.js                            # 대화형: 문장 입력 → 즉시 재생
node tools/audition.js 목록.txt                   # 텍스트 목록 배치 청취 (한 줄 = 한 항목)
```

항목마다 `Enter`=다음, `r`=다시 듣기, `f`=발음 교정(한글/가타카나 입력 → 사전 자동
저장 → 교정 반영해 바로 재청취), `d`=사전 항목 삭제(잘못 저장한 교정 되돌리기),
`m`=보류 메모, `q`=종료.

**f 교정의 저장처(마커 맥락 라우팅)**: `커피*`처럼 `*` 붙은 토큰 → `dict_jp.json`(일본어식,
마커/`--jp`일 때만 적용), 마커 없는 한글 어절 → `dict.json`(무조건 치환 — 가타카나 값 포함),
영어 단어 → `dict_en.json`(기본 한국어식 발음).

**가나 값의 길이 보정**: 마커 없는 한국어 맥락 가나(dict.json 값·직접 입력)는 규칙 엔진과
같은 받침 길이 보정을 받는다 — 어절 끝이나 자음 모라 앞의 ル·ム는 짧게(블랙→ブルレッ의 ル),
キェ류의 ェ는 연장. 일본어식(`*` 마커·dict_jp 경유)만 엔진 기본 길이 그대로. ン은 ㄴ/ㅇ을
구별할 수 없어 기본값 유지 — ン 길이가 어색하면 한글 재표기(dict.json 한글 값)로 교정.

**모라 길이 직접 지정(`<초>`)**: 자동 보정으로도 부족하면 모라 직후에 `<초>`를 붙여 그
모라의 길이(초)를 직접 지정할 수 있다 — `"블랙": "ブル<0.02>レッ'"`처럼 ル 뒤에 붙이면
자동 보정(0.032초)보다 우선해 0.02초로 짧아진다. 일본어식 가나에도 쓸 수 있고(그때도 항상
우선 적용), 값에서만 제거되고 실제 가나 출력엔 남지 않는다.

**보류 메모(m)**: 지금 수단(한글 재표기·가나 값·`'` 핵·`/` 구 분리)으로 해결되지 않는
문제 — 음소 길이·피치 수치 튜닝이 필요한 것 등 — 는 `보류목록.md`에 쌓인다.
쌓이면 검토해 ① 규칙 승격 ② 사전 등록 방법 고안 ③ 기획서 10장
(인토네이션·길이 손튜닝) 재개를 판단한다. 같은 유형의 오류가 3번 이상 반복되면
사전 대신 규칙을 고칠 대상이다.
