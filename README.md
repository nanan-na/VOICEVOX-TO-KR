# 한국어 → VOICEVOX 합성 도구

한국어 문장을 자체 규칙 엔진으로 **가타카나 + 악센트 마크업**으로 바꾼 뒤, 내 PC에서
돌아가는 로컬 VOICEVOX ENGINE(`localhost:50021`)으로 발음시키는 도구입니다.
웹 UI와 명령줄(CLI) 두 가지 방식으로 쓸 수 있습니다.

> 코드 주석에 나오는 "기획서 x.x"는 별도로 관리하는 설계 문서의 장 번호입니다.

## 시작하기

### 설치

1. [Node.js](https://nodejs.org/) LTS 버전을 설치해 주세요.
2. [VOICEVOX](https://voicevox.hiroshiba.jp/)를 설치해 주세요 (무료). 홈페이지에서 받는
   일반 VOICEVOX만 설치하면 되고, 엔진이 함께 들어 있어 따로 받으실 필요는 없습니다.
3. 이 저장소를 `Code → Download ZIP`으로 내려받아 압축을 풀어 주세요 (또는 `git clone`).
4. `start.bat`을 더블클릭하시면 됩니다.

외부 npm 패키지를 쓰지 않으므로 `npm install` 같은 준비 과정은 필요 없습니다.

### 실행

`start.bat`을 더블클릭하면 **엔진 확인 → 서버 실행 → 브라우저 열기**까지 자동으로
이어집니다. VOICEVOX를 기본 경로가 아닌 곳에 설치하셨다면, VOICEVOX를 먼저 한 번
실행해 두시면 `start.bat`이 그대로 이어받아 동작합니다.

직접 실행하고 싶으시면 `node server.js`를 켠 뒤 브라우저에서
`http://127.0.0.1:8300`으로 접속하시면 됩니다.

## 웹 UI 사용법

문장을 입력하고 **[변환] → [▶ 재생]**만 누르셔도 충분합니다. 아래는 화면에 보이는
순서대로 각 기능을 자세히 정리한 설명입니다.

### 상단바

- **화자**: 드롭다운에서 목소리를 고릅니다. 같은 화자의 스타일(감정 등)은 서브메뉴로
  묶여 있고, 화자·스타일마다 프로필 아이콘이 표시됩니다. 마지막에 고른 화자는 저장되어
  다음에 다시 켜셔도 그대로 유지됩니다.
- **외래어 일본어식**: 체크하면 문장 전체의 영어·외래어를 일본어 발음으로 읽습니다
  (`커피` → `コオヒイ`). 문장 전체가 아니라 단어 하나만 바꾸고 싶으시면, 이 토글 대신
  아래 **① 입력**의 `*` 마커를 쓰시면 됩니다.
- **엔진 상태**: 초록 점이면 VOICEVOX에 정상 연결된 상태입니다. 빨간 점이면 VOICEVOX가
  꺼져 있다는 뜻이니, 켜신 뒤 **[다시 연결]**을 눌러 주세요.

### ① 입력

- **마커 넣기**: 특정 단어만 일본어식으로 읽고 싶을 때, 그 단어 뒤에 `*`를 붙입니다.
  예를 들어 `커피* 한잔`이라고 쓰면 `커피`만 일본어식(`コオヒイ`)으로 읽고, 나머지는
  평소처럼 한국어식으로 읽습니다.
- **가나 직접 입력**: 규칙 변환을 거치지 않고 원하는 발음을 바로 넣고 싶으시면, 가타카나를
  그대로 적으시면 됩니다. `アンニョンハセヨ`처럼 전부 가나로 쓰셔도 되고, 문장 일부에만
  섞어 쓰셔도 그대로 인식됩니다.
- **[변환]** (또는 `Ctrl+Enter`): 입력한 문장을 가나로 바꿉니다. 타이핑하는 동안 자동으로
  변환되지는 않습니다 — 실수로 계속 재생되는 것을 막기 위해, 버튼을 눌러야만 반영됩니다.
- **[▶ 재생]**: 변환된 결과를 소리로 들려줍니다.
- **[wav 저장]**: 변환된 결과를 wav 파일로 내려받습니다.
- **자동 재생** (체크박스): 켜 두시면 [변환]을 누르는 즉시 재생까지 이어집니다.
- **[보류 메모]**: 지금 가진 수단(사전 등록·가나 수정)으로는 바로 고치기 애매한 발음을
  만났을 때 한 줄 남겨 두는 기능입니다. 입력창이 열리면 메모를 적고 [기록]을 누르시면
  `보류목록.md`에 쌓이고, 나중에 몰아서 검토하실 수 있습니다.

### ② 변환 결과

- 어절마다 원래 한글과 변환된 가나가 나란히 표시됩니다.
- 입력을 고치면 `⚠ 입력 변경됨` 배너가 뜨고 결과가 흐려집니다. 지금 보이는 것이 이전
  변환 결과라는 뜻이니, **[다시 변환]**을 누르시면 최신 입력으로 갱신됩니다.
- **어절을 클릭**하면 교정 팝업이 열립니다. `원본 변환(규칙)`과 `현재 사전값`을 비교하며
  새 발음을 입력하고 **[저장 및 재생]**을 누르시면, 사전에 저장되어 바로 반영됩니다
  (다음부터는 같은 단어에 자동으로 적용됩니다). 저장 위치는 audition 도구와 같은 방식으로
  자동 결정됩니다 — `*`가 붙은 단어는 dict_jp, 일반 한글은 dict, 영어 단어는 dict_en입니다.
- 재생하는 동안에는 지금 발음되는 어절이 순서대로 강조 표시됩니다.

### ③ 가나 마크업 편집

변환된 가나를 직접 손봐서 억양과 길이를 세밀하게 조정하실 수 있습니다.

| 기호 | 의미 | 예시 |
|---|---|---|
| `'` | 악센트 핵 (강세가 걸리는 자리) | `ブル'レッ` |
| `/` | 억양구 분리 (쉼 없이 억양만 끊음) | `チョン/ガヨ` |
| `<초>` | 그 모라의 길이를 초 단위로 직접 지정 | `ブル<0.02>レッ` (ル를 0.02초로) |

고치신 뒤에는 **[편집한 가나로 재생]**을 눌러야 반영됩니다 (아직 안 누르신 상태면
`● 미적용` 배지가 뜹니다). 가타카나가 낯설다면 **[가나 키보드 ▾]**를 펼쳐, 한글 발음
라벨이 붙은 가상 키보드로 입력하실 수 있습니다.

### 사전 관리

화면 하단의 "사전 관리"를 펼치면 지금까지 쌓인 예외 발음을 직접 확인하고 수정하실 수
있습니다.

- `dict.json` / `dict_en.json` / `dict_jp.json` 세 탭이 있으며, 각각 한글 발음, 영어 발음,
  일본어식 전용 발음을 담고 있습니다.
- 검색창(🔍)으로 항목을 찾아 수정하거나 삭제할 수 있습니다.
- 하단 입력창(키 → 값)으로 새 항목을 직접 추가할 수 있습니다.

## 명령줄(CLI) 도구

웹 UI 없이 터미널에서 바로 쓰고 싶으실 때 사용합니다.

### 기본 변환·합성

VOICEVOX(엔진 포함)를 먼저 실행하신 뒤:

```
node k2v.js "안녕하세요"                        # → out.wav
node k2v.js "뭐 해?" --out hi.wav --speaker 3   # 화자·출력 지정
node k2v.js "까치" --dry                        # 변환 결과만 (엔진 불필요)
node k2v.js "meeting 하자" --dry                # 영어 → 콩글리시 발음
node k2v.js "커피* 한잔" --dry                  # * 마커 → 그 단어만 일본어식 (コオヒイ)
node k2v.js "커피 마시자" --jp --dry            # 전역 일본어식 토글
node k2v.js --speakers                          # 화자 목록
```

영어 단어는 `dict_en.json`(관용 표기) → 약어 letter-name → 번들 발음사전
(`data/cmudict.json`, CMUdict) + 외래어 표기법 순서로 한글로 바꿔 기존 규칙 파이프라인에
태웁니다. 일본어식(`*` / `--jp`)일 때는, 한글 단어는 `dict_jp.json`을 조회하고 영어 단어는
CMUdict를 가타카나로 자동 변환합니다.

### 테스트

```
node test.js                # 골든 테스트 + 악센트 규칙 검증 (엔진 불필요)
node tools/probe-moras.js   # 엔진 확장 모라(クァ·ファ·キェ류 등) 지원 실측 (엔진 필요)
```

### 청취·사전 축적 (audition)

발음을 귀로 확인하면서 예외사전(dict.json)을 채워 가는 대화형 CLI입니다 (엔진 필요):

```
node tools/audition.js                            # 대화형: 문장 입력 → 즉시 재생
node tools/audition.js 목록.txt                   # 텍스트 목록 배치 청취 (한 줄 = 한 항목)
```

항목마다 다음 키를 쓸 수 있습니다 — `Enter`=다음, `r`=다시 듣기, `f`=발음 교정(한글/
가타카나 입력 → 사전 자동 저장 → 교정 반영해 바로 재청취), `d`=사전 항목 삭제(잘못
저장한 교정 되돌리기), `m`=보류 메모, `q`=종료.

#### f 교정의 저장처 (마커 맥락 라우팅)

- `커피*`처럼 `*`가 붙은 토큰 → `dict_jp.json` (일본어식, 마커/`--jp`일 때만 적용)
- 마커 없는 한글 어절 → `dict.json` (무조건 치환 — 가타카나 값 포함)
- 영어 단어 → `dict_en.json` (기본 한국어식 발음)

#### 가나 값의 길이 보정

마커 없는 한국어 맥락의 가나(dict.json 값·직접 입력)는 규칙 엔진과 같은 받침 길이 보정을
받습니다 — 어절 끝이나 자음 모라 앞의 ル·ム는 짧게(블랙 → `ブルレッ`의 ル), キェ류의 ェ는
연장됩니다. 일본어식(`*` 마커·dict_jp 경유)만 엔진 기본 길이를 그대로 씁니다. ン은 ㄴ/ㅇ을
구별할 수 없어 기본값을 유지하므로, ン 길이가 어색하면 한글 재표기(dict.json 한글 값)로
교정하시면 됩니다.

#### 모라 길이 직접 지정 (`<초>`)

자동 보정으로도 부족할 때는 모라 바로 뒤에 `<초>`를 붙여 그 모라의 길이(초)를 직접
지정할 수 있습니다. `"블랙": "ブル<0.02>レッ'"`처럼 ル 뒤에 붙이면 자동 보정(0.032초)보다
우선하여 0.02초로 짧아집니다. 일본어식 가나에도 쓸 수 있으며(그때도 항상 우선 적용),
값에서만 제거되고 실제 가나 출력에는 남지 않습니다.

#### 보류 메모 (`m`)

지금 수단(한글 재표기·가나 값·`'` 핵·`/` 구 분리)으로 해결되지 않는 문제 — 음소 길이·피치
수치 튜닝이 필요한 것 등 — 는 `보류목록.md`에 쌓입니다. 쌓이면 검토하여 ① 규칙 승격
② 사전 등록 방법 고안 ③ 인토네이션·길이 손튜닝 재개를 판단합니다. 같은 유형의 오류가
3번 이상 반복되면, 사전 대신 규칙을 고쳐야 할 대상입니다.

| `src/loanword.js` | ARPABET→한글(외래어 표기법)/가타카나 렌더러 |
| `src/accent.js` | 악센트 핵(`'`)·억양구(`/`) 자동 생성 |
| `src/voicevox.js` | ENGINE API 호출·길이 보정·WAV 결합 |
| `src/books.js` | 사전 3권 로드/저장/라우팅/값 검증 + 어절 분해 (audition·웹 공유) |
| `src/hold.js` | 보류 메모 기록 (audition `m`·웹 공유) |
| `k2v.js` | CLI 엔트리 |
| `server.js` | 웹 UI 서버 (내장 http, 정적+API, 의존성 없음) |
| `web/` | 웹 UI (index.html·app.js·style.css·kana-keyboard.js) |
| `start.bat` | 원클릭 실행 (엔진→서버→브라우저) |
| `config.json` | 화자 style_id·JP·자동 재생 설정 |
| `test.js` | 골든 테스트 fixture |
| `units.json` | 숫자 단위 사전 (사람이 편집) |
| `dict.json` | 예외사전 (사람이 편집, 쓸수록 축적) |
| `dict_en.json` | 영어→한국어 관용 표기 (audition `f`로 축적) |
| `dict_jp.json` | 일본어식 읽기 — `*`/`--jp` 전용 |
| `data/cmudict.json` | CMUdict 발음사전 번들 (기계 변환, 편집 안 함) |
